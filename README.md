## Security Best Practices for Your XRP

No wallet — including OKX Wallet — is automatically safe. Security is a set of habits, not a product feature. Here are the practices that actually matter:

**Treat your 12-word backup phrase like cash**

Write it on paper. Store it offline. Consider a metal backup plate for fire and water resistance. Never store it in a password manager, cloud note, email, or photo. If someone gets your backup phrase, they control your XRP — and no one, not even OKX, can help you recover it.

**Enable biometric authentication**

OKX Wallet supports fingerprint ID and facial recognition on mobile, and passkeys on desktop. Use them. A strong password is fine, but biometrics add a layer that's harder to phish.

**Test with small amounts first**

Before moving your entire XRP balance to a new wallet, send 20 XRP as a test. Confirm it arrives. Then send the rest. This catches address errors and configuration problems before they become catastrophic.

**Be careful with dApp connections**

OKX Wallet's security monitoring flags malicious domains and high-risk tokens, but no filter is 100% accurate. Before connecting your wallet to any dApp, verify the URL. Stick with pre-approved dApps shown in the wallet's Discover section when possible. Avoid using the manual dApp browser for unfamiliar sites.

**Keep your app updated**

When OKX pushes a wallet update, install it. Updates often include security patches for newly discovered vulnerabilities.

**Don't engage with airdropped tokens**

Scammers send tokens to random wallet addresses hoping you'll interact with them — clicking a malicious token's contract can drain your wallet. OKX flags high-risk tokens, but treat any unsolicited token deposit with suspicion.
